# セッション圧縮手順（stream-summarize）

コーディングエージェントの会話履歴を、後続ターンで使える要約に圧縮する。

## 目的

- ユーザーが何を達成しようとしているか
- 確定した技術判断・実装方針
- 変更したファイルとその要点
- 未解決の課題・次のアクション

## 圧縮の原則

1. **事実を残し、冗長なやり取りは捨てる** — 挨拶、試行錯誤の過程、重複説明は省略する
2. **コードは要点のみ** — 全文コピーせず、パス・関数名・変更の意図を記述する
3. **時系列より構造** — 話題ごとにまとめ、読み手が文脈を復元できるようにする
4. **不確かな情報は推測と明記** — 確定していない内容は「未確認」「要確認」と書く
5. **ツール出力は要約** — 長いログ・diff は結論と影響範囲に圧縮する

## 出力フォーマット

次のセクション見出しを使う（内容が無いセクションは省略可）。

- `## Goal` — ユーザーの目的
- `## Decisions` — 確定した判断
- `## Changes` — ファイル・実装の変更要点
- `## Open` — 未完了・ブロッカー
- `## Next` — 推奨される次のステップ

## イテレーション時の注意

- Current State がある場合は **マージ更新** する（上書きではなく統合）
- 新しい Input Chunks の情報を Current State に取り込み、重複を除去する
- 最終イテレーションでは、上記フォーマットに沿った **完結した要約** を Next State として出力する
