# Summarize-process 圧縮手順

分析フェーズと要約フェーズの 2 段でセッション履歴を圧縮する。

## Goal

コーディングエージェントの長い会話ログを、後続ターンで作業を継続できる要約に落とす。

## 圧縮の原則

1. **目的と現状** — 何を実装・調査していたか、今どこまで進んでいるか
2. **決定事項** — 採用した設計・却下した案と理由
3. **変更ファイル** — パスと変更の要点（推測で補完しない）
4. **未解決** — ブロッカー、未テスト、次の一手
5. **言語** — 元ログと同じ言語で書く（日本語ログなら日本語）

## 出力形式

- Markdown の見出し構造（`##`）で整理する
- メタ発言（「要約しました」等）は書かない
- コードブロックは必要最小限に留める
